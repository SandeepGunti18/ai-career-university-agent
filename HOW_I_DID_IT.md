# HOW I DID IT

## Step by Step

1. I selected a real problem many students face: choosing universities.
2. I built the project using Node.js for simplicity and easy execution.
3. I created separate tool functions for knowledge retrieval and insights.
4. I added input validation to prevent bad user entries.
5. I added try/catch error handling for reliability.
6. I improved explainability by showing tools used and reasoning.
7. I tested multiple scenarios like domestic and international choices.

## Problems Faced

- Understanding exact Level 3 expectations
- Structuring the repo professionally
- Improving reliability after evaluator feedback

## What I Learned

- AI tools need reliability, not just features.
- Explainable systems build user trust.
- Good project structure matters during reviews.
